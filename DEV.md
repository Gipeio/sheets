# Core Philosophy

Programming is not about knowing syntax.

Programming is about:

```
problem
↓
model
↓
algorithm
↓
code
```

Syntax is **the last step**, not the first.

The goal is to train the brain to think:

```
input
process
output
```

---

# The Real Skill of Programming

Good developers are good at:

```
1 understanding problems
2 decomposing systems
3 isolating bugs
4 iterating small solutions
```

Not memorizing code.

---

# Problem Solving Framework

Always solve problems with the same structure.

```
1 define the goal
2 define inputs
3 define outputs
4 design the transformation
5 implement
6 test
```

Example:

```
Goal: generate random number

Input: max value
Process: random(max)
Output: number
```

---

# Learning Loop

Every skill is built using the same loop.

```
read
↓
implement
↓
modify
↓
break
↓
fix
↓
repeat
```

Never stop at reading.

# Daily Practice System

Minimum effective practice:

```
30-60 minutes per day
```

Structure:

```
1 small exercise
1 new concept
1 small project
```

Example routine:

```
day 1
build simple API

day 2
connect button to API

day 3
send parameters

day 4
call python script

day 5
display results
```

---

# How to Build Projects

Always build in **iterations**.

```
version 1 → minimal working version
version 2 → add feature
version 3 → improve structure
```

Never build the final system first.

Example:

```
v1
button → random

v2
input → random

v3
python simulation

v4
graph
```

---

# Debugging Method

Debugging is a structured process.

```
1 reproduce bug
2 isolate component
3 log internal state
4 test assumptions
5 modify
```

Golden rule:

```
never guess
always observe
```

Tools:

```
console.log
network inspector
error stack traces
```

---

# Documentation Usage Strategy

Use documentation in this order.

```
1 official docs
2 minimal examples
3 experiment locally
```

Do not copy full solutions.

Instead:

```
read example
close documentation
rebuild from memory
```

Always think in **systems**.

A web application is:

```
client
↓
request
↓
server
↓
response
```

Or more explicitly:

```
UI
↓
API
↓
logic
↓
data
```

---

# Code Retention Strategy

To remember code, use this loop.

```
write
↓
rewrite from memory
↓
modify
↓
rewrite again
```

Repetition builds internal patterns.

---

# Project Architecture Thinking

Every project has the same layers.

```
interface
logic
data
```

Example web app:

```
HTML
↓
JavaScript
↓
Node API
↓
Python simulation
```

Separation is critical.

---

# The Minimal Stack Strategy

Avoid learning too many tools.

Choose a minimal stack.

Example:

```
HTML
JavaScript
Node
Python
```

This stack can build **almost any scientific web tool**.

---

# Reading Code

To understand code:

```
1 identify entry point
2 follow data flow
3 map functions
4 test behavior
```

Never read line by line first.

Always search for:

```
main
routes
handlers
```

---

# Writing Code From Memory

Best technique for independence.

Workflow:

```
read example
↓
close source
↓
rewrite from scratch
↓
compare
↓
fix mistakes
```

Repeat until natural.

---

# Refactoring Method

Once code works, improve it.

Refactor by:

```
removing duplication
simplifying logic
splitting functions
renaming variables
```

Never refactor before the program works.

---

# Building True Autonomy

Autonomy comes from three habits.

```
1 build many small programs
2 debug your own code
3 read official documentation
```

Not from tutorials.

---

# The Developer Growth Rule

Growth comes from this loop:

```
build
↓
fail
↓
debug
↓
understand
↓
repeat
```

The more cycles completed, the stronger the developer.

---

# Final Principle

The goal is not:

```
writing perfect code
```

The goal is:

```
being able to build anything step by step
```

Small steps create complex systems.
